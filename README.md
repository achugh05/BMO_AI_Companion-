# BMO OS: Desktop AI Companion 🤖
**Ansh Chugh, Jace Gordon, Son Nguyen, Ryan Cheng | University of British Columbia Okanagan (UBCO)**

BMO OS is an offline-first, conversational AI agent running on a **Raspberry Pi 4**. Inspired by [Brenpoly's design](https://github.com/brenpoly), our team of four engineering students developed this as a course project to integrate **Electrical Engineering** principles with modern local AI.

---

## ✨ Features
* **100% Local Intelligence**: Powered by **Ollama** (Llama 3.2:1b) and **Faster-Whisper**. No cloud latency or API fees.
* **Push-to-Talk (PTT)**: Optimized for the Pi 4 CPU; uses the `Enter` key to speak, saving RAM and power by avoiding a constant wake-word listener.
* **Study Mode (CV)**: Uses the **IMX500 Pi AI Camera** to track focus. Smart RAM management evicts the LLM (`keep_alive: 0`) to provide maximum resources for computer vision tasks.
* **Chatting Mode**: Real-time conversation generated by **Llama 3.2:1b**, including time checks, web searches via **DuckDuckGo**, and photo capture.
* **Music Mode (Beta)**: Features a fully custom **Spotify** music playing UI to use BMO as a dedicated speaker.
* **Gaming Mode**: Uses **EmulationStation** to emulate retro games with a completely custom BMO UI theme.
* **Academic Integration**: Custom tools to fetch assignments and grades via the **Canvas API**.

---

## 🛠️ Hardware Requirements
* **Raspberry Pi 4**: 8GB RAM recommended (4GB functional but prone to lag).
* **USB Microphone**: For voice input.
* **3.5mm Speaker/Headphones**: For audio output.
* **Raspberry Pi AI Camera Module**: Used for Study Mode and Photos (Standard Pi Camera works but requires more processing power).
* **LCD Screen**: 800x480 display for BMO's face and UI.

## 🚀 Installation & Setup

### 1. Environment Configuration
BMO requires a specific virtual environment to manage its audio and vision drivers.

```bash
# Clone the repository
git clone [https://github.com/achugh05/BMO_AI_Companion-.git](https://github.com/achugh05/BMO_AI_Companion-.git)
cd BMO_AI_Companion-

# Create and activate the virtual environment
python -m venv bmo_env
source bmo_env/bin/activate

# Install dependencies
pip install -r requirements.txt

```
### 2. Install & Configure Ollama
BMO's "Brain" relies on a custom-configured version of **Llama 3.2:1b**.

1. **Install Ollama**:
   ```bash
   curl -fsSL [https://ollama.com/install.sh](https://ollama.com/install.sh) | sh
*Alternatively, download the installer from [ollama.com](https://ollama.com).*

```
#### Create the BMO Model
BMO needs a specific configuration to handle tool-calling. Ensure you have a `Modelfile` in your directory, then run:

```bash
ollama create json_llama_bmo -f ./Modelfile

```
### 3. Retro Gaming Setup
Navigate to the **BMO Theme** folder for emulation-specific instructions: 
* [View the BMO Theme Folder](./bmo-theme/)

---
## 🎮 Running BMO
Ensure your environment is active and the Ollama service is running in the background.

```bash
# Activate environment
source bmo_env/bin/activate

# Launch BMO
python bmo_companion.py

* **Talk**: Press `Enter` in the terminal to start speaking to BMO.
* **Toggle Fullscreen**: Press `BackSpace` to toggle the UI (useful for hiding the Pi taskbar).
* **Emergency Exit**: Press `Space` to end the program immediately.

```
